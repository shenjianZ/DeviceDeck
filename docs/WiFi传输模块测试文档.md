# WiFi 文件传输 Web 界面测试文档

> DeviceDeck v0.1.1 | WiFi Transfer Module
>
> 测试范围：电脑端桌面应用 WiFi 传输控制面板 + 手机端 Web 传输界面

---

## 一、测试前置条件

| 条件 | 说明 |
|------|------|
| 电脑端 | 启动 DeviceDeck 桌面应用，进入「文件传输」页面，切换到 WiFi 模式 |
| 手机端 | 与电脑连接同一局域网（同一 WiFi 或同一网段） |
| 浏览器 | 手机端使用 Chrome / Safari / Firefox 等现代浏览器 |
| ADB 环境 | 电脑端已安装 ADB 或应用内置 ADB 可用 |

---

## 二、电脑端控制面板

### TC-D-01 启动 WiFi 传输服务

| 项目 | 内容 |
|------|------|
| 前置 | WiFi 传输服务处于停止状态 |
| 步骤 | 1. 点击「启动 WiFi」按钮 |
| 预期 | 按钮变为「停止 WiFi」，页面显示 QR 码、访问 URL、8 位连接码 |
| 验证 | QR 码图片正常渲染；URL 格式为 `http://<IP>:37210`；连接码为 8 位字符 |

### TC-D-02 停止 WiFi 传输服务

| 项目 | 内容 |
|------|------|
| 前置 | WiFi 传输服务运行中 |
| 步骤 | 1. 点击「停止 WiFi」按钮 |
| 预期 | 按钮变回「启动 WiFi」，QR 码/URL/连接码隐藏，状态显示「已停止」 |
| 验证 | 手机端已打开的页面不再能访问 API |

### TC-D-03 复制 URL 和连接码

| 项目 | 内容 |
|------|------|
| 前置 | WiFi 传输服务运行中 |
| 步骤 | 1. 点击 URL 旁的复制按钮 2. 点击连接码旁的复制按钮 |
| 预期 | 系统剪贴板分别包含对应的 URL 和连接码文本 |

### TC-D-04 QR 码扫码跳转

| 项目 | 内容 |
|------|------|
| 前置 | WiFi 传输服务运行中 |
| 步骤 | 1. 用手机相机或扫码工具扫描 QR 码 |
| 预期 | 手机浏览器打开 `http://<IP>:37210?token=<8位连接码>` |
| 验证 | 手机端 Web 界面自动填入连接码并完成验证，直接进入主界面 |

### TC-D-05 重复启动服务

| 项目 | 内容 |
|------|------|
| 前置 | WiFi 传输服务已运行 |
| 步骤 | 1. 再次点击启动（或在其他窗口触发） |
| 预期 | 服务保持运行，返回当前状态（不重复创建服务器），QR 码/URL/连接码不变 |

## 三、手机端认证页

### TC-A-01 Token 自动填充验证（扫码进入）

| 项目 | 内容 |
|------|------|
| 前置 | 通过 QR 码扫码打开 URL（URL 中包含 `?token=xxx`） |
| 步骤 | 1. 扫码后浏览器打开页面 |
| 预期 | 连接码自动填入输入框，自动触发验证，直接进入主界面 |
| 验证 | 看到 Header 栏显示「DeviceDeck 已连接」 |

### TC-A-02 手动输入 Token 验证成功

| 项目 | 内容 |
|------|------|
| 前置 | 手机浏览器直接输入 URL 打开页面（不带 token 参数） |
| 步骤 | 1. 在连接码输入框中输入电脑端显示的 8 位连接码 2. 点击「连接」按钮 |
| 预期 | 验证通过，页面切换到主界面，显示上传区域和文件列表 |

### TC-A-03 Token 验证失败

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 输入错误的连接码（如 `12345678`） 2. 点击「连接」 |
| 预期 | 输入框下方显示红色错误文本「连接码无效，请检查后重试」，输入框抖动动画 |
| 验证 | 页面停留在认证页，未进入主界面 |

### TC-A-04 连接按钮禁用状态

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 清空输入框 2. 观察连接按钮状态 |
| 预期 | 输入框为空时连接按钮 `disabled`（灰色不可点击） |
| 验证 | 输入任意字符后按钮变为可点击 |

### TC-A-05 Enter 键快捷验证

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 输入有效连接码 2. 按手机键盘的「回车/前往」键 |
| 预期 | 触发验证，等效于点击「连接」按钮 |

### TC-A-06 服务器不可达

| 项目 | 内容 |
|------|------|
| 前置 | 电脑端 WiFi 服务未启动，或手机与电脑不在同一网络 |
| 步骤 | 1. 输入任意连接码 2. 点击「连接」 |
| 预期 | 显示「无法连接到服务器」错误提示，输入框抖动 |

### TC-U-04 文件夹上传

| 项目 | 内容 |

## 六、连接状态管理

### TC-C-01 连接状态指示器（正常）

| 项目 | 内容 |
|------|------|
| 前置 | 正常连接状态 |
| 预期 | Header 显示绿色脉冲圆点 + 文本「已连接」 |

### TC-C-02 心跳检测 — 自动断开检测

| 项目 | 内容 |
|------|------|
| 前置 | 手机端已连接，电脑端 WiFi 服务运行中 |
| 步骤 | 1. 在电脑端点击「停止 WiFi」停止服务 |
| 预期 | 约 5 秒内（下一次心跳），手机端：状态点变红 + 「已断开」，顶部出现红色断开提示条，上传区域变为禁用（半透明不可操作） |

### TC-C-03 自动重连

| 项目 | 内容 |
|------|------|
| 前置 | 手机端检测到断开连接 |
| 步骤 | 1. 在电脑端重新启动 WiFi 服务（注意 Token 会变化） 2. 等待手机端自动重连 |
| 预期 | 手机端：状态点变为黄色 + 「重连中…」，每 3 秒尝试重连 |
| 注意 | 重连会因 Token 变更而失败（旧 Token 失效），最终停留在断开状态。这是预期行为 — 用户需刷新页面重新扫码 |

### TC-C-04 主动断开连接

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 点击 Header 右上角的断开按钮（退出图标） |
| 预期 | 触发断开流程：若当前有文件正在上传，则弹出确认弹窗；若无上传任务，则直接断开并返回认证页 |

### TC-C-05 断开确认弹窗 — 上传中

| 项目 | 内容 |
|------|------|
| 前置 | 队列中有文件正在上传 |
| 步骤 | 1. 点击断开按钮 |
| 预期 | 弹窗显示「正在传输的文件将被取消」警告 |
| 验证 | 点击「断开」后：所有上传被中止，页面回到认证页 |

### TC-C-06 断开确认弹窗 — 无上传

| 项目 | 内容 |
|------|------|
| 前置 | 队列中没有正在上传的文件 |
| 步骤 | 1. 点击断开按钮 |
| 预期 | 直接执行断开，不显示确认弹窗 |

### TC-C-07 断开确认弹窗 — 取消

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 触发断开确认弹窗 2. 点击「取消」或点击弹窗外区域 |
| 预期 | 弹窗关闭，页面保持在主界面 |

### TC-C-08 断开后状态重置

| 项目 | 内容 |
|------|------|
| 步骤 | 1. 添加文件到队列 2. 点击断开并确认 |
| 预期 | 队列清空，Token 清空，页面回到认证页，输入框为空 |

## 九、端到端测试流程

### E2E-01 完整传输流程（最核心测试）

```
1. 电脑端：启动 DeviceDeck → 文件传输 → WiFi 模式 → 点击「启动 WiFi」
2. 手机端：扫描 QR 码
3. 手机端：验证自动通过，进入主界面
4. 手机端：选择 3 个文件 → 点击「全部上传」
5. 观察：队列依次上传，进度条实时更新
6. 观察：全部完成后 Toast 提示「3 个文件上传完成」
7. 观察：文件列表自动刷新，显示 3 个文件
8. 电脑端：查看 `tempdir/devicedeck-wifi-transfer/` 目录确认文件存在
9. 手机端：点击某个文件的下载按钮
10. 验证：文件下载成功，内容正确
11. 手机端：点击断开 → 确认
12. 验证：页面回到认证页
13. 电脑端：点击「停止 WiFi」
```

### E2E-02 大文件传输

```
1. 准备一个 100MB+ 的测试文件
2. 上传该文件
3. 验证：进度条平滑更新，上传过程中可以取消
4. 上传完成后下载该文件
5. 对比源文件和下载文件的 MD5/SHA256 校验和，确认一致
```
