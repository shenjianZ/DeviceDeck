# 文件传输模块测试文档

## 1. 测试目标

验证 DeviceDeck 文件传输模块在 USB/ADB 和 Wi-Fi 两种模式下的功能完整性、稳定性、错误处理、中文/特殊字符兼容性、大文件性能和 UI 交互体验。

本测试文档覆盖：

- USB 设备文件浏览、排序、选择、导航、新建目录、删除。
- USB 上传、单文件下载、目录下载、批量下载。
- ADB 流式传输进度、速度、大小显示。
- 中文、空格、单引号等特殊路径兼容。
- Wi-Fi 传输服务启动/停止、Token 校验、二维码访问、上传、下载。
- 大文件/大目录传输内存风险。
- 前端状态、通知、错误提示和并发行为。

## 2. 测试环境

### 2.1 基础环境

- Windows 10/11。
- Android 设备 1 台，开启 USB 调试。
- ADB 可用，优先使用应用内置 ADB。
- 设备存储路径建议使用 `/sdcard/Download/DeviceDeckTest`。
- 手机和电脑处于同一局域网，用于 Wi-Fi 传输测试。

### 2.2 推荐测试文件

在电脑本地准备：

- `small.txt`：小文本文件。
- `中文 文件.txt`：中文和空格文件名。
- `quote's file.txt`：包含单引号文件名。
- `large-1gb.bin`：约 1GB 文件。
- `nested-folder/`：多级目录，包含中文文件、空目录、小文件、大文件。

在 Android 设备端准备：

- `/sdcard/Download/DeviceDeckTest/中文 文件.txt`
- `/sdcard/Download/DeviceDeckTest/quote's file.txt`
- `/sdcard/Download/DeviceDeckTest/nested/深层目录/a.txt`
- `/sdcard/Download/DeviceDeckTest/large-1gb.bin`

## 3. 自动化验证命令

每次修改文件传输模块后至少运行：

```powershell
rtk cargo test
rtk pnpm test:unit
rtk pnpm build
```

运行目录：

- `rtk cargo test` 在 `src-tauri` 目录。
- `rtk pnpm test:unit` 和 `rtk pnpm build` 在项目根目录。

提交前还需要运行 GitNexus 变更检测：

```powershell
gitnexus_detect_changes(scope: "all")
```

## 4. USB 文件浏览测试

### 4.1 设备选择

步骤：

1. 启动应用，进入“文件传输”页面。
2. 保持 Android 设备已连接且授权 USB 调试。
3. 查看设备下拉框。
4. 选择在线设备。

预期结果：

- 下拉框只展示在线设备。
- 选择设备后自动加载默认路径 `/sdcard`。
- 无设备时显示空状态和扫描按钮。
- 扫描按钮可重新刷新设备列表。

### 4.2 目录列表加载

步骤：

1. 选择设备。
2. 打开 `/sdcard/Download`。
3. 打开不存在路径，例如 `/sdcard/not-exist-xxxx`。
4. 打开无权限路径，例如 `/data`。

预期结果：

- 正常目录显示文件和目录列表。
- 不存在路径显示错误通知，不应清空当前有效目录状态。
- 无权限路径显示错误通知和建议。
- 加载期间显示 loading 状态。

### 4.3 路径输入和面包屑导航

步骤：

1. 在路径输入框输入 `/sdcard/Download` 并回车。
2. 点击面包屑中的 `/`。
3. 点击面包屑中的中间路径段。
4. 点击上级目录按钮。

预期结果：

- 回车和跳转按钮都能加载目标目录。
- 面包屑路径正确拆分。
- 上级目录在 `/` 时不继续跳转。
- 路径输入框随当前目录同步更新。

### 4.4 排序

步骤：

1. 在包含多种文件和目录的目录中点击名称列。
2. 点击大小列。
3. 点击修改时间列。
4. 每列重复点击两次。

预期结果：

- 每列可在升序/降序之间切换。
- 目录始终排在文件前面。
- 排序不会改变选中项。
- 空 size/modified 不导致页面报错。

### 4.5 选择与快捷键

步骤：

1. 单击文件行。
2. Ctrl/Command 单击多个文件。
3. 按 Ctrl/Command + A。
4. 按 Esc。
5. 选中文件后按 Delete。

预期结果：

- 单击只选中当前项。
- Ctrl/Command 单击可多选/取消。
- 全选选择当前列表全部项。
- Esc 清空选择。
- Delete 触发删除确认，不直接删除。

## 5. USB 文件操作测试

### 5.1 新建文件夹

步骤：

1. 在 `/sdcard/Download/DeviceDeckTest` 右键空白区域。
2. 点击“新建文件夹”。
3. 输入 `新建 测试目录`。
4. 回车确认。

预期结果：

- 设备端创建目录成功。
- 创建后刷新当前目录。
- 中文和空格目录名可正常创建。
- 输入为空时不提交。
- Esc 可取消输入。

### 5.2 删除文件

步骤：

1. 选择普通文件。
2. 右键点击删除。
3. 取消确认。
4. 再次删除并确认。

预期结果：

- 取消确认时不删除。
- 确认后设备端文件被删除。
- 删除后刷新当前目录。
- 删除失败时显示错误通知。

### 5.3 删除目录

步骤：

1. 选择测试目录。
2. 执行删除并确认。

预期结果：

- 目录和子内容被删除。
- 不允许删除 `/`、`/sdcard`、`/storage` 等保护路径。
- 删除后当前目录刷新。

### 5.4 批量删除

步骤：

1. 多选 3 个文件或目录。
2. 右键选择“删除选中项”。
3. 确认删除。

预期结果：

- 所有选中项按顺序删除。
- 任一项失败时显示失败通知。
- 完成后清空选择。
- 当前目录最终刷新。

## 6. USB 上传测试

### 6.1 文件选择器上传

步骤：

1. 进入 `/sdcard/Download/DeviceDeckTest`。
2. 点击“发送到设备”。
3. 选择 `small.txt`、`中文 文件.txt`、`quote's file.txt`。

预期结果：

- 文件按顺序上传。
- 上传成功后显示成功通知。
- 当前目录刷新并显示新文件。
- 中文、空格、单引号文件名可正常上传。
- Android 媒体索引刷新不影响上传结果。

### 6.2 拖拽上传

步骤：

1. 将多个本地文件拖入文件列表区域。
2. 观察上传过程和目录刷新。

预期结果：

- 拖拽 hover 有视觉反馈。
- 多文件按顺序上传，不并发打爆 ADB。
- 上传时不阻塞导航、选择、刷新等无关操作。
- 上传完成后进度短暂停留并自动消失。

### 6.3 大文件上传

步骤：

1. 上传 `large-1gb.bin`。
2. 观察内存、进度、速度、完成状态。

预期结果：

- 应用内存不随文件大小线性增长。
- 进度显示百分比、速度、已传输/总大小。
- 上传失败时显示 ADB 错误。
- 成功后设备端文件大小一致。

## 7. USB 下载测试

### 7.1 单文件下载

步骤：

1. 右键设备端 `small.txt`。
2. 点击“保存到电脑”。
3. 选择本地保存目录。

预期结果：

- 文件保存到本地目录。
- 文件内容一致。
- 进度显示真实已传输字节、总大小、速度。
- 完成后显示成功通知和输出路径。

### 7.2 中文/特殊字符文件下载

步骤：

1. 下载 `中文 文件.txt`。
2. 下载 `quote's file.txt`。

预期结果：

- 文件名不乱码。
- 路径包含空格、中文、单引号时下载成功。
- 本地文件名与设备端文件名一致。

### 7.3 大文件下载

步骤：

1. 下载设备端 `large-1gb.bin`。
2. 观察内存、进度、速度。

预期结果：

- 使用流式写盘，不把整个文件读入内存。
- 失败时本地残留文件被清理或明确可识别。
- 成功后文件大小一致。

### 7.4 目录下载

步骤：

1. 右键设备端 `nested-folder`。
2. 点击“保存文件夹到电脑”。
3. 选择本地保存目录。

预期结果：

- 优先使用设备端 tar 流式拉取。
- Windows 本地解包中文文件名不乱码。
- 不出现重复顶层目录，例如 `nested-folder/nested-folder/...`。
- 空目录、深层目录、中文目录均被保留。
- tar 不可用时回退递归逐文件下载。

### 7.5 批量下载

步骤：

1. 多选多个文件和目录。
2. 右键点击“保存选中项”。
3. 选择本地保存目录。

预期结果：

- 选中项按顺序下载。
- 文件和目录都能处理。
- 完成后清空选择。
- 单项失败不应导致 UI 卡死。

## 8. 传输进度测试

### 8.1 已知总大小进度

步骤：

1. 上传或下载普通文件。
2. 观察进度条。

预期结果：

- 显示文件名。
- 显示 `已传输 / 总大小 - 百分比 - 速度`。
- 百分比在完成前不超过 99%。
- 完成时为 100%。

### 8.2 未知总大小进度

步骤：

1. 模拟设备端 `stat` 或 `du` 不可用。
2. 执行下载。

预期结果：

- UI 使用非确定进度条。
- 至少显示已传输字节和速度。
- 完成时正常移除进度项。

### 8.3 多任务进度

步骤：

1. 连续上传多个文件。
2. 连续下载多个文件。

预期结果：

- 每个任务有独立进度 ID。
- 完成事件短暂停留后自动清理。
- 不同任务不互相覆盖进度。

## 9. Wi-Fi 传输测试

### 9.1 服务启动与停止

步骤：

1. 切换到 Wi-Fi 模式。
2. 点击开启 Wi-Fi 传输。
3. 再点击停止。

预期结果：

- 启动后显示服务运行状态。
- 显示访问 URL、Token、二维码。
- 停止后状态更新为未启动。
- 重复启动时返回当前状态，不重复绑定端口。
- 端口被占用时显示错误通知。

### 9.2 URL 和二维码

步骤：

1. 启动 Wi-Fi 服务。
2. 用手机扫码访问。
3. 手动复制 URL 打开。

预期结果：

- IPv4 URL 格式正确。
- IPv6 URL 使用 `[addr]` 包裹。
- 二维码包含 token 参数。
- 手机端可打开页面。

### 9.3 Token 校验

步骤：

1. 使用正确 token 访问。
2. 使用错误 token 访问。
3. 不带 token 访问。

预期结果：

- 正确 token 可进入主界面。
- 错误 token 显示 Invalid token。
- 未授权接口返回 401 或无数据。

### 9.4 手机上传到电脑

步骤：

1. 手机端选择小文件上传。
2. 手机端选择中文文件名上传。
3. 上传同名文件两次。
4. 上传大文件。

预期结果：

- 上传使用分块写入，不整文件读入内存。
- 中文文件名保留。
- 危险字符被替换。
- 同名文件不会覆盖，自动生成后缀。
- 大文件上传在限制内成功，超过限制时失败明确。
- 电脑端收到 `transfer://file-received` 事件。

### 9.5 手机下载电脑端已接收文件

步骤：

1. 手机端刷新文件列表。
2. 下载小文件。
3. 下载中文文件名文件。
4. 下载大文件。

预期结果：

- 文件列表按名称排序。
- 下载使用流式响应，不整文件读入内存。
- `Content-Disposition` 支持 UTF-8 文件名。
- 文件内容和大小一致。

## 10. 错误与边界测试

### 10.1 设备断开

步骤：

1. 传输过程中拔掉 USB。
2. 执行刷新或上传。

预期结果：

- 显示 ADB/设备连接错误。
- UI 不永久卡在传输中。
- 后续重新连接后可继续使用。

### 10.2 权限不足

步骤：

1. 访问 `/data` 或其他无权限目录。
2. 尝试删除系统保护路径。

预期结果：

- 显示权限或路径错误。
- 不执行危险删除。

### 10.3 本地保存路径异常

步骤：

1. 选择不存在或无权限保存目录。
2. 下载文件。

预期结果：

- 可创建普通不存在目录。
- 无权限时显示写入失败。
- 不产生 UI 卡死。

### 10.4 路径特殊字符

测试字符：

- 中文。
- 空格。
- 单引号 `'`。
- 括号。
- `#`、`&`。

预期结果：

- USB 远端 shell 命令正确转义。
- 文件可浏览、上传、下载、删除。
- Wi-Fi 文件名净化只替换危险字符，不破坏中文。

## 11. 性能与稳定性测试

### 11.1 大文件

测试：

- 上传 1GB 文件。
- 下载 1GB 文件。
- Wi-Fi 上传 1GB 文件。
- Wi-Fi 下载 1GB 文件。

预期结果：

- 内存占用稳定，不随文件大小线性增长。
- 进度持续更新。
- 传输失败可恢复 UI 状态。

### 11.2 大目录

测试：

- 下载包含 1000+ 文件的目录。
- 下载包含多级嵌套和中文文件名的目录。

预期结果：

- tar 流式下载优先成功。
- 回退递归下载时不 OOM。
- 文件数量和目录结构一致。

### 11.3 长时间运行

测试：

- Wi-Fi 服务开启 1 小时。
- 多次上传/下载。
- 多次启动/停止服务。

预期结果：

- 状态一致。
- 不重复绑定端口。
- 无明显内存泄漏。

## 12. 回归测试清单

修改文件传输模块后，至少确认：

- 设备扫描和其他页面未被破坏。
- 旧的设备页 `pushDeviceFile` 仍可用。
- 截图、安装 APK、按键、shell 命令不受传输辅助函数影响。
- 设置页、日志页、镜像页可正常编译和测试。
- 中英文 i18n 文件 JSON 格式合法。

## 13. 验收标准

满足以下条件才可认为文件传输模块通过：

- USB 浏览、上传、下载、删除、新建文件夹全部通过。
- 中文、空格、单引号路径全部通过。
- 大文件传输无明显内存暴涨。
- 目录下载无中文乱码、无重复目录层级。
- Wi-Fi 服务启动/停止、上传、下载、Token 校验全部通过。
- `rtk cargo test` 通过。
- `rtk pnpm test:unit` 通过。
- `rtk pnpm build` 通过。
- GitNexus `detect_changes` 已检查并确认影响范围符合预期。
